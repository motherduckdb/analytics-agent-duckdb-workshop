Why DuckDB for Agentic?

An agent needs a database it can reach for in milliseconds, without provisioning anything. DuckDB fits the loop.

In-process
Runs inside the agent's Python process. No network hop, no connection pool.
Easy to install
pip install duckdb. One binary, zero dependencies. Agent is shippable.
Fast feedback
Sub-second queries on millions of rows → many tool calls per turn stay cheap.
Reads anything
CSV, Parquet, JSON, S3, Iceberg, Delta — one SQL surface for the model.
