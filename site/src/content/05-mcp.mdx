## MCP Separates Tools from the Agent

Your tools work — but they're bundled in your script. MCP **separates tools from the agent** so anyone can use them.

| Before (Part 4) | After (Part 5) |
|-----------------|----------------|
| Agent + tools in one script | Tools in MCP server |
| Only you can use it | Any MCP client connects |
| You pick the model | Client picks the model |

<Tip>MCP is the USB-C port. Your agent is the device. The server has no model — just Python functions.</Tip>

@mcp.tool()
def search_tables(keyword: str) -> str:
    """Search table and column descriptions stored via COMMENT ON."""
    con = get_con()
    pattern = f"%{keyword}%"
    tables = con.execute(
        "SELECT table_name, comment FROM duckdb_tables() WHERE comment ILIKE ?",
        [pattern]).fetchall()

## PydanticAI as MCP Client

An agent that discovers tools **from the server**:

<CopyBlock label="agent_mcp.py" text={`import asyncio
from pydantic_ai import Agent
from pydantic_ai.toolsets.fastmcp import FastMCPToolset

async def main():
    toolset = FastMCPToolset("mcp-server/server.py")
    agent = Agent(
        "anthropic:claude-sonnet-4-20250514",
        system_prompt=open("skill.md").read(),
        toolsets=[toolset],
    )
    async with agent:
        result = await agent.run("Revenue by venue for 2025, and who are the top 5 customers?")
        print(result.output)

asyncio.run(main())`} />

**Zero tools defined in the agent.** PydanticAI discovered them from the MCP server.

## Simple Tools vs Smart Tools

Our tools are <Badge color="#16AA98">deterministic</Badge> — pure Python. Some tools need a <Badge color="#FFDE00">specialized model</Badge>:

| Tool type | Model on server? | Example |
|-----------|:---:|---------|
| Deterministic | No | `revenue_by_venue()` |
| API call | No | `check_inventory()` |
| Specialized AI | **Yes** | `analyze_video()` &rarr; Gemini Vision |
| Embedding search | **Yes** | `semantic_search()` |

<Tip>Client model = generalist (reasoning). Server model = specialist (vision, embeddings). Most tools don't need a model.</Tip>

## Checkpoint

**You stopped building the agent. You started serving capabilities.**

- **Skill** = flexibility (instructions the LLM follows)
- **Tool** = determinism (same input &rarr; same output)
- **MCP** = distribution (build once, any client connects)
- **COMMENT ON** = traps become warnings

From local to cloud: **MotherDuck MCP** is the managed version of what you just built.
