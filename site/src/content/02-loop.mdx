## <Badge color="#6FC2FF">Era 3</Badge> Build the Loop Yourself

~35 lines of Python. The agentic loop is just a `while` loop:

<CopyBlock label="Create agent_loop.py" text={`from openai import OpenAI  # works with Ollama
import duckdb, json

client = OpenAI(base_url="http://localhost:11434/v1", api_key="ollama")
MODEL = "gemma4:26b"
con = duckdb.connect("duckoffee.duckdb", read_only=True)

tools = [{"type": "function", "function": {
    "name": "query_duckdb",
    "description": "Execute a read-only SQL query on the Duckoffee database.",
    "parameters": {"type": "object", "properties": {"sql": {"type": "string"}}, "required": ["sql"]},
}}]

def run_tool(name, args):
    if name == "query_duckdb":
        try:
            rows = con.execute(args["sql"]).fetchall()
            cols = [d[0] for d in con.description]
            return json.dumps({"columns": cols, "rows": rows[:50]}, default=str)
        except Exception as e:
            return json.dumps({"error": str(e)})

def ask(question):
    messages = [{"role": "user", "content": question}]
    while True:
        response = client.chat.completions.create(model=MODEL, tools=tools, messages=messages)
        choice = response.choices[0]
        messages.append(choice.message)
        if choice.finish_reason != "tool_calls":
            return choice.message.content
        for call in choice.message.tool_calls:
            args = json.loads(call.function.arguments)
            print(f"  tool: {call.function.name}: {args.get('sql', '')[:70]}")
            result = run_tool(call.function.name, args)
            messages.append({"role": "tool", "tool_call_id": call.id, "content": result})

print(ask("What are the top 5 best-selling products?"))`} />

## Now Watch It Fail

The agent will confidently pick the **wrong** table. Try these:

<CopyBlock label="Trap questions" text={`What's the average order value?`} />

<Tip>The agent will likely query `raw_orders.order_total` and return ~17,000 — because raw_orders stores amounts in **cents**, not euros. The correct answer is 169.90 euros from the `orders` table.</Tip></invoke>

<CopyBlock label="More traps" text={`Which venue has the highest total revenue?
How many repeat customers do we have?
What's the daily revenue trend for the last 3 months?`} />

Common failures:
- **Cents vs euros**: `raw_orders` is 100x too high
- **Column names**: `raw_orders.customer` vs `orders.customer_id`, `store_id` vs `location_id`
- **Enum values**: no `'repeat'` in customer_type — it's `'returning'`
- **Dead-end tables**: `daily_revenue` only has 5 of 20 venues, stops at 2025

## What's Solved, What's Left

The agentic loop and text-to-SQL on clean schemas? **Solved.** opencode, Claude Code, or ~35 lines of Python handle it.

What **breaks it** is your real data — traps the model can't see:

| The Trap | What Happens |
|----------|-------------|
| `raw_orders` amounts in cents | Revenue is 100x too high |
| `daily_revenue` table is stale | Missing 3 venues, no 2026 data |
| `customer_type` = `'returning'` not `'repeat'` | Zero results |
| `raw_orders.customer` has no `_id` suffix | Silent wrong joins |

**Next:** we add **semantic context** — a skill file + database comments — so the model knows which tables to trust and what the columns actually mean.
