## What is DuckDB?

DuckDB is an in-process analytical database. Think SQLite, but for analytics. It queries files directly — no server needed.

<CopyBlock label="Quick CLI test" text={`duckdb duckoffee.duckdb -c "SHOW TABLES;"`} />

---

## Python is Home

This is PyCon — let's use Python.

<CopyBlock label="Create part1.py" text={`import duckdb

con = duckdb.connect("duckoffee.duckdb", read_only=True)

# What tables do we have?
con.sql("SHOW TABLES").show()

# How big is the data?
con.sql("""
    SELECT
        (SELECT count(*) FROM orders) AS orders,
        (SELECT count(*) FROM customers) AS customers,
        (SELECT count(*) FROM order_items) AS line_items,
        (SELECT count(*) FROM products) AS products,
        (SELECT count(*) FROM locations) AS venues
""").show()`} />

<CopyBlock label="Run it" text="python3 part1.py" />

## Explore the Venues

<CopyBlock label="Venues and cities" text={`con.sql("""
    SELECT location_name, city, venue_type
    FROM locations
    ORDER BY city, location_name
""").show()`} />

6 venues across 3 cities: Amsterdam, Berlin, Istanbul. Each is a different type — Main Hall, Workshop Room, Cafe, Rooftop.

## Revenue by Venue

<CopyBlock label="Revenue query" text={`con.sql("""
    SELECT l.location_name, l.city,
           count(*) AS orders,
           round(sum(o.order_total), 2) AS revenue
    FROM orders o
    JOIN locations l ON o.location_id = l.location_id
    GROUP BY ALL
    ORDER BY revenue DESC
""").show()`} />
