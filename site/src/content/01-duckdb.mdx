## Read Anything, One Line

DuckDB queries CSV/Parquet/JSON — locally or **straight from S3** — with a single line.

<CopyBlock label="CSV from a public S3 bucket" text={`FROM 'https://us.data.motherduck.com/who_ambient_air_quality/csv/who_ambient_air_quality_database_version_2024.csv'
LIMIT 5;`} />

<CopyBlock label="What extensions are loaded?" text={`FROM duckdb_extensions() WHERE loaded ORDER BY extension_name;`} />

<Tip>No `CREATE TABLE`, no `COPY`, no schema file. `httpfs` auto-loads. This is why DuckDB is the data scientist's duct tape.</Tip>

## Python is Home

This is PyCon — let's use Python. Same engine, same SQL.

<CopyBlock label="Create part1.py" text={`import duckdb

con = duckdb.connect()                       # in-memory session
con.sql("ATTACH 'duckoffee.duckdb'")         # same concept as ATTACH 'md:' for MotherDuck

con.sql("""
    SELECT
        (SELECT count(*) FROM duckoffee.orders) AS orders,
        (SELECT count(*) FROM duckoffee.customers) AS customers,
        (SELECT count(*) FROM duckoffee.order_items) AS line_items,
        (SELECT count(*) FROM duckoffee.products) AS products,
        (SELECT count(*) FROM duckoffee.locations) AS venues
""").show()`} />

<CopyBlock label="Run it" text="uv run python part1.py" />

You should see: **120k orders, 12k customers, 500k line items, 25 products, 20 venues.**

## Revenue by Venue

<CopyBlock label="Revenue query" text={`con.sql("""
    SELECT l.location_name, l.city,
           count(*) AS orders,
           round(sum(o.order_total), 2) AS revenue
    FROM orders o
    JOIN locations l ON o.location_id = l.location_id
    GROUP BY ALL
    ORDER BY revenue DESC
""").show()`} />

<Exercise title="Write your own query" minutes={3}>
  Try querying the `customers` table. How many returning vs new customers are there? (Hint: the column is `customer_type` with values `'new'` and `'returning'`.)
</Exercise>

## What about MotherDuck?

<div style={{ display: "flex", alignItems: "center", gap: 16, marginBottom: 10 }}>
  <img src={asset("/motherduck-logo.svg")} alt="MotherDuck" style={{ height: 40, width: "auto" }} />
</div>

**DuckDB, supercharged.** MotherDuck is the serverless analytics platform built on DuckDB — run the same SQL from your laptop to a cloud data warehouse to customer-facing applications, with zero infrastructure to manage.

<CopyBlock label="One line to connect" text={`ATTACH 'md:';`} lang="sql" />

That single statement connects your local DuckDB session to MotherDuck's cloud. Query local files and cloud tables in the same SQL — the engine splits work between your machine and the cloud automatically.

<Tip>Everything you built today with a local `.duckdb` file works the same way on MotherDuck — skill files, COMMENT ON, PydanticAI tools, MCP servers. Just swap the connection string.</Tip>
